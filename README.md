# 万有引力模拟程序

这是一个使用现代OpenGL（3.3核心模式）和Bullet物理引擎实现的多体万有引力模拟程序。程序模拟了多个球体在三维空间中的万有引力相互作用，并提供了实时3D可视化。

## 功能特性

- **物理模拟**：使用Bullet物理引擎进行精确的刚体动力学模拟
- **万有引力**：实现真实的万有引力公式（F = G * m1 * m2 / r²）
- **3D可视化**：使用现代OpenGL进行高质量渲染，支持Phong光照模型
- **交互控制**：支持第一人称相机控制，可自由观察模拟场景
- **随机生成**：程序随机生成多个具有不同质量、位置和颜色的球体
- **坐标轴显示**：显示3D坐标轴（X-红，Y-绿，Z-蓝）辅助空间定位
- **实时重置**：支持按R键重置模拟状态

## 技术栈

- **图形API**：OpenGL 3.3（核心模式）
- **物理引擎**：Bullet Physics
- **数学库**：GLM（OpenGL Mathematics）
- **窗口管理**：GLFW
- **扩展加载**：GLEW
- **着色器语言**：GLSL 330

## 构建说明

### 依赖项

1. **OpenGL**（3.3或更高版本）
2. **Bullet Physics**（2.89或更高版本）
3. **GLFW**
4. **GLEW**
5. **GLM**

### 构建步骤

```bash
# 克隆项目
git clone <your-repo-url>
cd gravity-simulation

# 创建构建目录
mkdir build && cd build

# 配置项目（根据你的系统调整）
cmake ..

# 构建项目
make

# 运行程序
./GravitySimulation
```

### Windows用户注意事项

Windows用户可能需要手动配置Bullet库路径。

## 使用方法

### 启动程序
程序启动后会创建一个窗口，显示随机生成的球体在引力作用下的运动。

### 交互控制

- **W/A/S/D**：前后左右移动相机
- **空格键**：向上移动相机
- **左Shift键**：向下移动相机
- **鼠标移动**：控制视角方向
- **R键**：重置模拟（停止所有球体并恢复到初始位置）
- **ESC键**：退出程序

### 模拟参数

在`Simulation.cpp`中可以调整以下参数：
- `numSpheres`：球体数量（默认50个）
- `posDist`：球体初始位置范围（默认±5.0单位）
- `massDist`：球体质量范围（默认1.0-100.0）
- `G`：万有引力常数（默认6.67430e-5f，已放大以便观察）

## 实现细节

### 物理模拟
1. 使用Bullet Physics创建动态世界
2. 每个球体对应一个刚体（btRigidBody）
3. 在每帧更新中计算所有球体对之间的引力
4. 使用Bullet的`stepSimulation`进行物理更新

### 图形渲染
1. **球体渲染**：
   - 使用细分球体网格（32个扇区，16个堆栈）
   - 实现Phong光照模型（环境光+漫反射+镜面反射）
   - 使用统一颜色（不存储每个顶点的颜色）

2. **坐标轴渲染**：
   - 简单的线条渲染
   - 禁用深度测试以确保始终可见

3. **相机系统**：
   - 第一人称相机
   - 欧拉角控制偏航和俯仰
   - 透视投影矩阵

### 内存管理
- 使用智能指针（`std::unique_ptr`）管理资源
- 实现RAII（Resource Acquisition Is Initialization）原则
- 移动语义支持，禁用拷贝构造/赋值

## 性能优化

1. **着色器优化**：
   - 预取常用uniform位置
   - 缓存uniform位置以减少API调用

2. **渲染优化**：
   - 使用索引缓冲对象（EBO）
   - 顶点数据批量上传
   - 深度测试启用

3. **物理优化**：
   - 刚体去激活禁用（保持活跃状态）
   - 惰性计算（仅在需要时更新）

## 已知问题

1. 大量球体时性能可能下降（考虑使用空间划分优化）
2. 引力计算复杂度为O(n²)，球体数量多时会成为瓶颈
3. 球体碰撞处理较简单，实际应用中可能需要更复杂的碰撞响应
